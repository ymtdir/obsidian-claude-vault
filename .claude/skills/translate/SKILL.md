`00_INBOX/` に投入された海外（日本語以外）の記事クリップを、1記事=1ノートで日本語要約・翻訳し、`40_TRANSLATIONS/` に保存する。末尾には必ず元記事URLを記載する。元素材は触らない。

## 手順

### 1. 対象ファイルの特定

- 引数でファイル名が指定されていれば `00_INBOX/<ファイル名>` のみ対象
- 引数がなければ `00_INBOX/` 直下の `.md` を全件列挙し、1件ずつ順に処理
- 対象が空なら「INBOX は空です」と伝えて終了

### 2. ソース読み込みと元記事URLの特定

- ファイル本文を Read で読み込む
- 本文中の元記事URLを自動抽出する
  - `http://` / `https://` で始まるURL、Markdownリンク、`Source:` / `URL:` / `原文:` のような見出し・ラベル行を探す
  - 候補が複数ある場合は、記事本体と思われるURL（ファイル冒頭の出典記載、最も上位に出てくるURL等）を優先
- 見つからない／判別できない場合は、処理開始前にユーザーにURLを確認する
- ユーザーが「URL 不明」「URL なしで進める」と回答した場合は **処理を中止する**（URL なしでのノート保存は行わない。本 skill は元記事 URL の明記が必須方針のため）

### 3. 要約と翻訳

- 記事の主要な主張・論点・結論を把握する
- 日本語で要約する。逐語訳ではなく読みやすさ重視
- 元記事の章立て（見出し構造）は基本的に踏襲する
  - 章見出し中の一般語（例: `Introduction` / `Summary` / `概述` / `最佳实践`）は日本語化する（「はじめに」「まとめ」など）
  - 見出し内の **技術用語・固有名詞**（`Virtual DOM`、`Fiber`、`goroutine`、`QUIC` など）は原語のまま残す
- 固有名詞・技術用語・コード識別子・製品名・ライブラリ名は原語のまま残す
- コードブロックはそのまま保持し、必要ならコメントや説明文のみ日本語化

### 4. 40_TRANSLATIONS にノート作成

- ファイル名は元記事タイトルを日本語訳したもの（例: `How React Works.md` → `Reactの仕組み.md`）
  - **副題は圧縮可**（主題のみでも OK。副題は本文冒頭の見出しで扱う）
  - **OS 予約文字**（`/`、`:`、`?`、`*`、`|`、`\`）は `-` や `_` に置換する（例: `HTTP/3` → `HTTP-3`）
  - 区切り記号は **全角 `：`** を標準とする
- 既存ファイルと衝突する場合は括弧で補足（例: `Reactの仕組み（2024年版）.md`）
- プロパティ（YAML frontmatter）は使わない
- ファイル名がタイトルになるので `# 見出し` は不要。本文の見出しは `##` から使い、その下の区分けに `###` を使う
- `[[wikilink]]` は張らない（`40_TRANSLATIONS/` はリンク付け不要方針）
- **本文末尾に `## 出典` セクションを設け、元記事URLを必ず記載する**
  - フォーマット: Markdown リンク形式でリスト記載（例: `- [タイトル](URL)`）
  - **リンクテキストは元記事の原題**（翻訳前タイトル）を使う。日本語訳題はファイル名側で扱い、出典は原典性を優先
  - 著者・サイト名・公開日が分かる場合は次行以降にリスト項目として追加（例: `- 著者: 名前`、`- サイト: 名前`、`- 公開日: YYYY-MM-DD`）

### 5. レポート

- 作成した `40_TRANSLATIONS/` ノート名を提示する
