# Obsidian Vault — Claude Code ルール

日本語で応答すること。

## Vault 構造

| フォルダ        | 役割                                                                                                  | ファイル命名規則         | リンク戦略                               |
| --------------- | ----------------------------------------------------------------------------------------------------- | ------------------------ | ---------------------------------------- |
| `00_INBOX/`     | 一時的な調査メモ・クリップの投げ入れ先。未整理の状態で置かれる                                        | 自由（内容がわかる名前） | 不要                                     |
| `10_DAILY/`     | デイリーノート。朝に今日の目標を記入し、夜に今日の成果・ふりかえりを記入。`/daily` でAIがまとめを生成 | `YYYY-MM-DD.md`          | 学び・気づきから `20_NOTES/` へリンク    |
| `20_NOTES/`     | 調査・学習ノート。フォルダ分けせずフラットに保存する                                                  | トピックを端的に表す名前 | ノート間を `[[wikilink]]` で積極的に接続 |
| `30_DOCUMENTS/` | 共有用ドキュメント・記事執筆用。外部出力するためリンク付け不要                                        | 自由                     | リンクを挿入しない                       |
| `80_SECURE/`    | 機密情報・個人情報など git 管理外のノート。中身は `.gitignore` で除外                                 | 自由                     | 不要                                     |
| `90_TEMPLATES/` | Obsidian のテンプレートファイル置き場                                                                 | 用途を表す名前           | 不要                                     |
| `99_ASSETS/`    | 画像などのアセット                                                                                    | —                        | —                                        |

## ツェッテルカステン原則

- **1ノート = 1アイデア**: `20_NOTES/` の各ノートは単一のトピックを扱う
- **リンクで接続**: ノート間は `[[wikilink]]` で接続する。タグやフォルダではなくリンクが構造を作る
- **自律性**: 各ノートは単独で読んで意味が通じるように書く

## デイリーノートのワークフロー

`10_DAILY/` のノートは以下の構成で整理する：

1. **今日の目標**: 1日の初めにユーザーが記入。今日やるタスクを箇条書きで記載
2. **今日の成果**: 1日の終わりにユーザーが記入。実際にやったことを自由形式で記載
3. **ふりかえり**: 1日の終わりにユーザーが記入。今日の目標・成果を照らし合わせ、できたこと・できなかったこと・反省点などを自由形式で記載
4. **まとめ**: `/daily` 実行時にAIが生成。ふりかえりを読んで良かった点と改善点をコーチのようにコメント

`/daily` 実行時、AIはまず今日の目標・今日の成果・ふりかえりの文章を添削してからまとめを生成する。

## ノート作成ルール

- Obsidian のプロパティ（YAML frontmatter）は使わない
- ファイル名がタイトルになるので `20_NOTES/` では `# 見出し` は不要。見出しは `##` から使い、その下の区分けに `###` を使う
- 関連ノートへの `[[wikilink]]` は本文の文脈中に自然に埋め込む（末尾の一覧にまとめない）
